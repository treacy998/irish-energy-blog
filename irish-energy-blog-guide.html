1

Activate the virtual environment

Do this once per terminal session. You'll see (venv) in your prompt.

cd ~/dev/blog/irish-energy-blog
source venv/bin/activate
2

Download the data

Go to sem-o.com/market-data/dynamic-reports → ETS Market Results. Download yesterday's DAM CSV. Drop it into data/ — no renaming needed.

Important

Only SEM-DA files work. Do not use SEM-IDA1, SEM-IDA2, or SEM-GB files — those are different markets and the pipeline will reject them.

3

Run the pipeline

python pipeline/run_daily.py

Auto-detects the most recent MarketResult_SEM-DA_*.csv, prints the delivery date, then prompts for a post title:

Post title [I-SEM Daily Briefing — 13 May 2026]:
# Press Enter for the default, or type a custom headline
# e.g. "Evening spike hits €187/MWh as wind collapses"

To use a specific file instead of auto-detecting:

python pipeline/run_daily.py data/MarketResult_SEM-DA_PWR-MRC-D+1_20260513100000.csv
4

Write your commentary

Open site/content/daily/YYYY-MM-DD.md. Stats table and chart embeds are pre-filled. Write 2–3 paragraphs in the Commentary section.

5

Preview locally

cd site && ~/.local/bin/hugo server -D

Opens at localhost:1313. Hot-reloads on save. Ctrl+C to stop.

6

Publish

git add . && git commit -m "Daily briefing YYYY-MM-DD" && git push

Vercel auto-deploys. Live in ~60 seconds.

7

Post to LinkedIn

Upload the best chart PNG as an image. Write 2–3 sentences of insight. Put the blog link in the first comment, not the post body.

SEMO Data Format

PropertyDetail
DelimiterSemicolon ;
DecimalsEuropean comma: 161,70 = 161.70
LayoutWide/transposed — 48 values across one row, not 48 rows
Rows 1–7Metadata (auction name, date, FX rate)
Data blocksRepeating 3-row groups: header / timestamps / values
TimestampsUTC. SEM day = 22:00Z → 21:30Z (23:00–22:30 Irish)
Delivery dateAuction date + 1 day (D+1 in filename)

Data Blocks

BlockWhatUsed?

Index prices GBPSame in sterlingNo
Index volumesTraded volume per period (MWh)Not yet
Net positionCross-border net flow (MW)Not yet
Default blocksBaseload/peak product resultsNo
Timezone

The pipeline converts UTC → Irish time with a +1h offset. Correct for April–October (IST). In November–March (GMT), times show 1 hour ahead. Full DST fix using zoneinfo.ZoneInfo("Europe/Dublin") planned for October.

Writing Posts

Daily Post Frontmatter

---
title: "Evening spike hits €187/MWh as wind drops — 12 May 2026"
date: 2026-05-12
authors: ["Eoin"]
tags: ["daily-briefing", "DAM", "I-SEM"]
summary: "DAM prices averaged €94/MWh, peaking at €187/MWh at 17:30."
draft: false
---

The title is set at the prompt when you run the pipeline. The summary is auto-filled from stats and doubles as the SEO meta description.

Commentary Structure

¶1 — What happened

"DAM prices followed a typical demand-driven profile, averaging €94/MWh with an evening peak of €187/MWh at 17:30…"

¶2 — Why

"The steep evening ramp coincided with wind dropping to 14% of demand, forcing gas units up the merit order…"

¶3 — Context

"This was the highest evening peak this week, contrasting with Tuesday's flatter profile when wind exceeded 50%…"

Theme — Congo

Replaced PaperMod. Key features: card layout on list pages, ocean colour scheme, dark/light toggle, built-in search, table of contents, author + reading time on every post.

Config Files

All config lives in site/config/_default/. Do not create a site/hugo.toml — Hugo uses the directory instead.

hugo.toml

Base URL, permalinks, taxonomies, markup settings

params.toml

Colour scheme, layout, article/list display options

languages.en.toml

Site title, description, author name, social links

menus.en.toml

Navigation menu items and ordering

Colour Scheme

Edit params.toml:

colorScheme = "ocean"   # options: ocean, sapphire, azure, slate, fire, forest, neon, congo, rose, pink

Dark Mode

defaultAppearance = "light"   # "light" or "dark"
autoSwitchAppearance = true    # follows OS preference

Path Fix

If hugo isn't found, use ~/.local/bin/hugo. To fix permanently:

echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.bashrc && source ~/.bashrc

No module named pandasActivate the venv: source venv/bin/activate
Pipeline ignores my fileCheck filename contains SEM-DA. IDA1/IDA2/SEM-GB are rejected. Or pass the path explicitly.
No data for {date}Date in CSV doesn't match. Run python pipeline/process.py to see detected date.
FileNotFoundErrorNo MarketResult_SEM-DA_*.csv in data/. Download from SEMO.
Charts look wrongRun python pipeline/process.py to inspect parsed numbers.
